# Оценка времени разработки

## Общие параметры
- Рабочий день: 8 часов
- Запас на тесты и отладку: +20%

⸻

## 1. База данных для парсера

### Задачи:
- Проектирование схемы БД
- Создание таблиц (категории, товары, маркетплейсы, цены, история, логи)
- Настройка индексов и оптимизация
- Миграции и документация
- Настройка бэкапов

**Оценка:** 24 часа (3 дня)  
**С запасом (+20%):** 29 часов (3.6 дня ≈ **4 дня**)

⸻

## 2. Алгоритм парсера

### Задачи:
- Архитектура и настройка окружения
- Модуль получения списка товаров
- Парсеры для каждого маркетплейса (базовый + специфичные)
- Обработка статических и динамических страниц
- Обработка антибот-защиты
- Извлечение и валидация данных
- Сохранение в БД
- Планировщик задач (cron/scheduler)
- Мониторинг и логирование
- Обработка ошибок и retry-логика
- Конфигурация

**Оценка:** 80 часов (10 дней)  
**С запасом (+20%):** 96 часов (12 дней)

⸻

## 3. Фронтенд для Telegram-бота

### Задачи:
- Настройка проекта и интеграция с Telegram Web App API
- Онбординг (6 экранов: анимация, описание, форма, поиск, результаты, карточка)
- Главный экран
- Поиск товаров (категории, товары, анимация поиска, результаты, карточка)
- Навигация между экранами
- Обработка ошибок
- UI/UX компоненты
- Анимации
- Оптимизация и тестирование

**Оценка:** 120 часов (15 дней)  
**С запасом (+20%):** 144 часа (18 дней)

⸻

## 4. Бэкенд для Telegram-бота

### Задачи:
- Настройка проекта и архитектура
- Интеграция с Telegram Bot API
- API endpoints (аутентификация, онбординг, категории, товары, поиск)
- Бизнес-логика
- Работа с БД (пользователи + парсер)
- Валидация данных
- Обработка ошибок
- Безопасность
- Логирование и мониторинг
- Кэширование
- Оптимизация производительности
- Тестирование и документация API

**Оценка:** 80 часов (10 дней)  
**С запасом (+20%):** 96 часов (12 дней)

⸻

## 5. База данных для Telegram-бота

### Задачи:
- Проектирование схемы БД
- Создание таблиц (пользователи, согласия, история поисков)
- Настройка индексов
- Миграции и документация
- Настройка бэкапов
- Интеграция с БД парсера

**Оценка:** 16 часов (2 дня)  
**С запасом (+20%):** 19 часов (2.4 дня ≈ **3 дня**)

⸻

## 6. Дополнительные задачи

### Задачи:
- Настройка инфраструктуры (серверы, CI/CD, домены)
- Документация проекта
- Интеграционное тестирование
- Аудит безопасности
- Настройка аналитики

**Оценка:** 40 часов (5 дней)  
**С запасом (+20%):** 48 часов (6 дней)

⸻

## Примечания

- Оценки даны с учётом работы одного разработчика
- При работе команды время может сократиться за счёт параллельной разработки
- Время на парсер зависит от количества маркетплейсов и сложности их защиты
- Время на фронтенд зависит от сложности анимаций и требований к UI/UX
- Рекомендуется заложить дополнительный буфер на непредвиденные задачи (+10-15%)
